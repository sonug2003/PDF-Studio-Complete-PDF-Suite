Scope

This Privacy Policy applies to the Android application Complete PDF Studio (package name: com.completepdfstudio), including its PDF processing, conversion, annotation, and export features.

This policy covers how the app code accesses, processes, stores, and shares user data through app features.

If the legal entity name, contact email, or support details change, update this policy and your Google Play disclosures before release.

Information Collected or Processed

Based on the audited source code, the app may access and process the following categories of data when you use app features:

  • Files you select from your device (such as PDF files and selected images).
  • File contents of selected documents and images for local processing (for example, OCR, text extraction, conversion, merge/split, annotation).
  • User-entered document content and metadata (for example, added text, watermark text, PDF metadata fields, password fields).
  • Generated output files (for example, edited PDFs, extracted text files, exported images, generated HTML/text outputs).
  • Temporary cache copies of selected files used for processing workflows.
Data Category How It Enters the App Primary Purpose
User-selected PDFs/images System document pickers (OpenDocument/OpenMultipleDocuments) Local PDF tools and conversion workflows
Document text/metadata Read from selected files or entered by the user Edit, view, OCR, extraction, and metadata operations
Output files Created by app processing features Save, open, or user-initiated share/export
Local diagnostic warnings Internal logging for invalid page range parsing Debug/validation behavior in local app execution

Information Not Collected (Code-Audit Findings)

The audited code does not include logic for collecting or transmitting the following categories as part of an app backend service:

  • No user account registration, login, or profile backend.
  • No payment, subscription, or in-app billing integration.
  • No analytics SDK integration detected in source usage.
  • No ad SDK integration detected in source usage.
  • No crash-reporting SDK integration detected in source usage.
  • No push notification service integration detected.
  • No direct collection of device identifiers such as IMEI/Android ID in app code.
  • No contacts, SMS, call log, or location access in the audited app code.
The app source contains no direct HTTP client integration or explicit app-to-backend API request implementation.

How Information Is Used

Data is used only to perform user-invoked app functionality, including:

  • Reading selected files into app cache for processing.
  • Applying requested PDF operations such as edit, convert, merge, split, OCR, redact, annotate, and export.
  • Rendering previews and extracting text/links for in-app viewing functions.
  • Saving outputs to app storage and optionally to device Downloads when the user chooses to save/export.
  • User-initiated share actions through Android share intents.

Data Sharing and Data Selling

The audited app code does not implement automatic data selling or automatic sharing of your files/content to a developer backend.

Data may leave the app only in user-initiated cases, such as:

  • You choose Share and send a file through another app via Android share sheet.
  • You choose to open a link embedded in a PDF, which launches another app (for example, a browser).
User-initiated file transfer through Android share actions is treated as user-directed sharing, not data sale.

Storage Locations and Data Retention

The app stores data locally on your device in these locations:

  • App cache directory for temporary processing copies (for example, selected file cache and merge temp files).
  • App external files directory under a PDFStudio subfolder for generated outputs.
  • Optional user-visible Downloads/PDFStudio export location when you choose Save to Device.
Data Type Where Stored Retention Behavior
Temporary merge files App cache Explicitly deleted after merge operation completes.
Other cached file copies App cache Persist until app/system cache cleanup or app uninstall.
Generated output files App external files/PDFStudio Persist until user deletion or app uninstall.
User-exported files Downloads/PDFStudio Persist until user manually deletes them.

The manifest sets android:allowBackup="true". The repository does not define a custom backup policy file, so any platform-managed backup behavior depends on Android system behavior and user/device settings.

Data Security

  • Processing is primarily local to the device according to audited code paths.
  • File sharing uses Android FileProvider and requires a user action.
  • No direct app backend transmission code was identified in the source audit.
  • Password-protection features can be applied by users to PDF outputs as an optional document-level protection control.
No software can guarantee absolute security. Device security (lock screen, OS updates, malware hygiene) also affects protection outcomes.

International Data Transfers

The audited app code does not implement direct developer backend transfers. Therefore, there are no app-controlled international transfers to a developer server shown in the audited source.

If you intentionally share files to third-party apps or open external links from documents, those third-party services may process data under their own policies and jurisdictions.

Children's Privacy

The audited source does not implement a child-specific account model, child profiling logic, or dedicated child-directed data collection features.

If this app is later positioned as child-directed, additional compliance steps may be required under applicable child privacy laws and Google Play Families requirements.

User Rights and Controls

Because this app does not include an in-app account system or developer-hosted user profile backend in audited code, your primary controls are local device controls:

  • Delete files you imported or exported on your device.
  • Clear app data/cache using Android app settings.
  • Uninstall the app to remove app-private data from the device.
  • Control sharing by choosing whether to send files via external apps.

Account Deletion and Data Deletion

Account deletion is not applicable to the audited app code because no in-app account creation or sign-in system was found.

Data deletion is device-based in this app architecture:

  • Delete exported files from Downloads/PDFStudio or wherever you saved them.
  • Delete files from app storage by clearing app data or uninstalling the app.
Google Play account-deletion web-link requirements apply when an app supports account creation. No such flow was found in this audited codebase.
